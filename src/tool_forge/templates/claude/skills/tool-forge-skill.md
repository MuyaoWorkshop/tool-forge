# Tool-Forge Skill

## 名称
tool-forge

## 描述
系统化地创建和管理个人Python CLI工具的Skill

## 触发条件

当用户说：
- "帮我创建一个CLI工具"
- "新建命令行工具"
- "开发一个工具"
- "创建一个自动化脚本"
- 提到具体工具需求（如"JSON格式化工具"）

## 能力

### 1. 需求分析
- 理解用户想要什么工具
- 明确功能需求和约束
- 识别输入输出
- 推荐技术方案

### 2. 项目规划
- 选择合适的模板
- 设计项目结构
- 规划开发步骤
- 估算开发时间

### 3. 代码生成
- 基于模板生成项目
- 编写核心功能
- 实现CLI接口
- 添加错误处理

### 4. 质量保证
- 代码规范检查
- 最佳实践应用
- 性能优化
- 安全检查

### 5. 测试和文档
- 编写测试
- 生成文档
- 创建示例
- 验证功能

### 6. 发布管理
- Git版本控制
- 推送到GitHub
- 创建Release
- 工具注册

## 使用场景

### 场景1：从零创建工具
**用户**: "帮我创建一个JSON格式化工具"

**tool-forge**:
1. 需求分析：读取JSON、格式化、输出
2. 模板选择：basic-cli
3. 初始化：`python scripts/init_tool.py json-formatter -t basic-cli`
4. 实现：读取、解析、美化、输出
5. 测试：创建测试JSON
6. 发布：uv install、git push

### 场景2：基于现有工具改进
**用户**: "我需要一个pdf-split的改进版，支持加密PDF"

**tool-forge**:
1. 分析现有工具
2. 识别需要改进的部分
3. 创建新工具分支
4. 添加新功能
5. 测试和发布

### 场景3：批量创建工具
**用户**: "我需要一批数据处理工具"

**tool-forge**:
1. 识别共同模式
2. 创建通用模板
3. 批量生成工具
4. 统一测试和发布

## 工作流程选择

### 标准工作流
使用场景：
- 学习CLI开发
- 创建高质量工具
- 长期维护项目

时间：1.5-3小时

### 快速工作流
使用场景：
- 原型验证
- 一次性工具
- 简单需求

时间：25-45分钟

## 模板库

### basic-cli
**适用**：简单工具
- 单一功能
- 标准输入输出
- 无需配置

**示例**：json-formatter, text-counter

### advanced-cli
**适用**：复杂工具
- 多个子命令
- 配置文件支持
- 插件系统

**示例**：file-manager, task-runner

### specialized
**适用**：特定领域
- PDF处理
- JSON处理
- 文件操作

**示例**：pdf-split, json-cleaner

## 常见问题

**Q: 如何选择模板？**
A: 功能简单用basic-cli，复杂用advanced-cli，特定领域用specialized。

**Q: 需要多长时间？**
A: 标准工作流1.5-3小时，快速工作流25-45分钟。

**Q: 支持哪些Python版本？**
A: Python 3.8+

**Q: 如何发布工具？**
A: 通过GitHub + uv安装，不支持PyPI。

**Q: 可以商业化吗？**
A: 可以，但需要注意依赖许可证。

## 限制

- 不支持GUI工具
